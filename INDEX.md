## Operating Rhythm
- **Daily:** 60–90 minutes study or build. Log key outcomes in `04_Logs/Progress_Reports/` with an ISO date prefix.
- **Weekly:** Review progress, update project boards, cut a progress report.
- **Monthly:** Run a retrospective using the template in `04_Logs/Monthly_Reviews/`.

## Conventions
- Filenames start with `YYYY-MM-DD_` where relevant.
- Keep all project-level configuration in `PROJECT.yaml`.
- Every experiment gets a notebook and a short summary in `docs/`.
- Prefer plain Markdown for portability.
